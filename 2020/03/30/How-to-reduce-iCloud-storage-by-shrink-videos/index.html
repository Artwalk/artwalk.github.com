How to reduce iCloud storage by shrink videos

LeiHao's Blog
2020-03-30
Exiftool, FFmpeg, iCloud

When I gallery videos in Photos, it will show where and when did I take them

That is a wonderful feature to help me to recall my memory
But the original 4k takes too much space in iCloud

For example, a 2:30 video will take 1GB, even powered by HEVC (High-Efficiency Video Coding)
My 200G iCloud plan almost full

So I’m planning to compress videos a little bit to save some space.

But,

FFmpeg converts video will loss GPS even with metadata

Luckily, I found a way to do it

Tools

  • FFmpeg
  • Exiftool

Export original videos to a folder

The common way to drag videos from Photos to a folder will let Photos convert the video

So you have to select all your videos want to export and

File -> Export -> Export Unmodified Original For...

shrink each video

iPhone default Stream is
#0:0(und): Video: hevc (Main) (hvc1 / 0x31637668), yuv420p(tv, bt709)

so re-encoding to 10bit with -pix_fmt yuv420p10le does not help

Then you can copy & paste these code into Terminal and cd VideoFolder, then shrinkVideos

It will take a very long time to do the shrink

On my MacBook Pro (15-inch, 2018), it’s only 0.3x
But usually, the video will shrink 5x without losing too much quality

When all finished, shrunk videos will under shrunk/ folder

Then you can compare them, and delete original video first then import shrunk video

OK Finally I cannot bear the extremely slow speed, so I use my Nvidia 1080 GPU to accelerate, for more: CUDA GPU Accelerated h264/h265/HEVC Video Encoding with Staxrip
