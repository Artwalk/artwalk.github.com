Xcode 6.x cannot run on EI Capitan

Artwalk's Blog
2015-07-08
Programming

I watched WWDC and updated OS X to EI Capitan(10.11).
Everything seemed fine until I opened Xcode:

You can’t use this version of the application “Xcode” with this version of OS X.
You have “Xcode” 6.x.

Swift 2.0 is awesome, but according to Apple, if you want to submit an app which written by 2.0 to App Store, you need to wait until September.

Fortunately I found a hack command:

Which means you run Xcode from /dev/null and redirect output & error output to /dev/null in background.

Everything perfect in past several weeks.
