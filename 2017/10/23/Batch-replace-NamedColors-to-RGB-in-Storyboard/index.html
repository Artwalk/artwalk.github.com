Batch replace NamedColors to RGB in Storyboard

LeiHao's Blog
2017-10-23
Xcode, iOS

Updates: 2018-04-15

As the Xcode updated to 9.3, the NamedColors will cause an error instead of a warning; the shell doesn’t have to much value.

And there is a fast way to do it by click the color in Storyboard and touch the Hex Color # say “FFFFFF” after you chose the NamedColor.

NamedColors is a new feature that convenient for developers to use in Xcode
But it is only available on iOS 11, and it will cause app crash on iOS 10, 9…

The main idea is:

  • Use NamedColors in StoryBoard to avoid fill color every time to benefit the new feature.
  • Replace all NamedColors’ attributes to RGB in StoryBoard when building the project to prevent crash on an old iOS version.

Then, Xcode -> Project File -> Build Phases -> Add Build Phase -> Add Run Script

If you found any way to reverse it back to NamedColors, please leave a msg to me.

After all, once the NamedColors changed, all the color in the interface builder should be changed too.
