How to add Xib on StoryBoard

Artwalk's Blog
2016-10-18
Xcode, iOS

In the classic 1975 software project management book, The Mythical Man Month: Essays on Software Engineering, Fred Brooks states that, no matter the programming language chosen, a professional developer will write an average 10 lines of code (LoC) day.

StoryBoard is good for saving code lines and improve efficiency, as  strongly recommend in each WWDC and continually improve it.

I don’t understand why there are still lots of Team Leaders require their poor programmers to write the UIView in programmatic way.

Maybe they thought some CustomViews are highly reuse usable, and cope paste in StoryBoard would cause the redundant side effect.

So, The main idea of this solution is, cohesion the CustomViews to a Xib then add it on StoryBoard.

This need a few steps as for  didn’t support it directly:

  1. add a IBDesignable CustomView.Swift and CustomView.Xib
  2. set the CustomView.xib -> Placeholders -> File's Owner -> Custom Class to CustomView
  3. new a CustomView programmatically and load the root view from CustomView.Xib
  4. add the root view with equal width and height in both horizontal and vertical center on CustomView in init function
  5. now you can drag a View to StoryBoard, and set it’s class to CustomView which can generate a snapshot for preview

But the subviews of CustomView like UIButton can’t wire to ViewController as an @IBOutlet

Delegate or Block can solve this problem.

As for me, I suggest Block way, cause I used Delegate for half a year, and found

  • ViewController will have too many delegates
  • delegate = self have to written in viewDidLoad and sometimes =nil in viewDidDisappear

Block will have less code and more simpler.

Here is the example which may reduce your time.
