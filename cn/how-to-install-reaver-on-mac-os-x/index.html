如何在Mac OS X上安装reaver

发表于 2014-05-11 更新于 2020-12-13 Disqus：
本文字数： 886 阅读时长 ≈ 1 分钟

更新: 2017-05-24 09:03:10

对不起，这个帖子，我之前没有意识到它的**危险性。

去年 10 月我的网超期服役，我为新网需要的国庆假期过去的临时用的 wifi 重新 aver 了一下。

当我在 NAS 上播放开放的耶鲁课程-听音乐时，发现我的电视（安卓系统）有一个字幕，上面写着 “本网仅供我个人使用，你是不是有什么问题？”。

我被这句话吓了一跳，我的 NAS 有一个 PandoraBox 系统，我用它做桥接，平时我的电视不应该一个出现在他们的路由器上。很明显，我发现这个 wifi 是钓鱼的 wifi，我的电视被黑了。

所以，如果一些愚蠢的应用还在使用 http，我所有的数据都暴露了。连 https 都有MitM的问题。’的产品说 iOS，macOS 有一个登录信任证书在 Keychain，但 Android 是什么？有些设备有他们自己的后门。

至于我的电视，我觉得它用的是adb connect <TV IP地址>也就是 _Android Debug Bridge (adb) Wireless Debugging Over Wi-Fi_。

不过，我后来改了大部分密码。

所以，你可以试试技术探索的 reaver，但不能用它。

更新: 2015-07-27 13:14:32

我是在 2014-05-11 写的，不过晚了一年，现在用Homebrew就可以轻松安装 reaver。

感谢 Rustam 的建议

我总是在 vmware 中使用 beini，每次都会多插一个 USB 无线适配器到我的电脑上。

当我有 Mac 的时候，我只是把 beini 的 VMX 从 Windows 复制到 Mac 上，这意味着我仍然要使用无线适配器。

其实，minidwep或者inflator都是用Reaver这个强大的工具来 … 你懂的），而 Mac 是基于 unix 的系统，可以直接运行Reaver。

  1. 下载 reaver-osx.zip
    (小贴士：原来的 reaver-1.4 不能在 Mac 上直接使用make)
  2. 解压 reaver-osx.zip。

尽情享受吧，不要做坏事:-)
