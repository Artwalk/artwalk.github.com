如何刷 Cookbook

天哪，终于翻完了，1k 多页啊！！！
把里面 80% 的代码手打了一遍，发现了代码中的两个 BUG
另外有错误的，童鞋们可以在 Github 上 Pull Requests 给 oreillymedia/iOS7_Programming_Cookbook

适合人群：

一般来说，Cookbook 系列都被类比做字典，用到什么，查什么，没必要头到尾翻一遍

但刷一遍后，对相关知识体系搭建的好处是显而易见的

不适合:

  1. 小白
  2. iOS 资深开发者

小白们可以先把这两本刷了：
Head First iPhone and iPad Development
The Big Nerd Ranch Guide

方法：

  1. 首先要做好崩溃的准备 →_→
    想想当年多少童鞋背字典到 abandon 就跪了

  2. 选择重点的几章，即和你当前工作相关的章节一字不漏的看
    由于看的东东立即就能用到，所以效果比较好
    非代码部分是 原理 以及 tips，所以认真看下

  3. 到 50% 左右时，重点转向代码
    这时基本熟悉作者的叙述方式了，扫下标题，基本上就知道要干嘛，结合代码连蒙带猜，文字部分扫一下就行了

  4. 要懂得放弃
    背字典并不能让你说一口流利的英语，所以此书也无全部刷完的必要
    比如我看的时候，就嫌 iCloud 之类的太麻烦了，如果简单的同步数据，还不如直接搭个后台方便，所以就略过
    算下来正好符合 二八法则

  5. 结对
    自己单干的效果是没有小组学习高的
    不过你们可以看的不一样，比如我盆友看的就是 Python 的 Cookbook，达到相互促进的目的就行
    PS: 当我看了一大半时，才知道他看的是译文版的…

  6. 英语
    iOS 更新太快了，最新的都是英文的，7 还没翻译呢，8 的正式版就要发布了…
    So，学好英语吧，骚年
